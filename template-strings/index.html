Template Strings

Vicente Calfo, 
January 1st, 2021 · 4 min read

Introdução

Eu trabalhei no desenvolvimento de um projeto que tinha toneladas de javascript, um genome browser (é uma ferramenta muito útil para os bioinformatas e se parece com isso aqui) maneiríssimo, com uma penca de funcionalidades e consequentemente uma outra tonelada de informações de texto que eram mostradas ao usuário dinâmicamente a cada interação com a interface.

Ali eu percebi que a interpolação de strings em javascript poderia ficar caótica em pouco tempo, ainda mais se eu precisasse trabalhar com múltiplas linhas, injetando tags de HTML na página (DOM).

Mas eram outros tempos, não tinha Angular, React e nem tinha um monte de coisas legais que vieram com o ES6.

Então lá estava eu tentando lidar da maneira mais elegante e menos caótica possível com o diacho das interpolações.

Com as especificações do ECMAScript 2015 foi inserido as Template Strings. Uma funcionalidade que veio para deixar as coisas nada caóticas e bem mais elegantes.

Tudo que eu queria no meu passado mencionado acima.

O que é

As template strings são uma funcionalidade em javascript que nos permite criar interpolações de string sem a necessidade de utilizar o sinal de mais (+).

Múltiplas linhas

Em algum momento você vai acabar se deparando com a necessidade de interpolar strings que terão como conteúdo tags de HTML que precisarão ser inseridas dinamicamente na página (DOM).

Naquele projeto que eu mencionei lá no começo do artigo eu tive que fazer várias vezes isso. Eu sofri um pouco (terapia mode on).

Vamos pegar um exemplo simples como uma div que vai exibir uma mensagem com a marcação de HTML um pouco mais complexa. Iremos imaginar que essa mensagem terá duas colunas sendo que uma delas será um ícone e na outra a mensagem de texto.

Vamos exercitar a criatividade?

Se tem uma coisa empolgante no processo de desenvolvimento de códigos é a construção de soluções criativas e elegantes para resolver determinados problemas/tarefas.

Então, vamos imaginar que eu queira fazer uma lista não ordenada para mostrar ao usuário uma série de nomes de projeto para nossa aplicação fictícia utilizada nos códigos de exemplo.

Usando a interpolação sem template strings poderíamos escrever a função para gerar o HTML necessário seguindo a abordagem abaixo.

Abriríamos a tag da lista e faríamos um loop no array de projetos para ir concatenando as strings interpoladas com o nome do projeto a cada item e no final fecharíamos a lista para construír o HTML necessário.

Agora vamos usar um pouco da criatividade para resolver esse mesmo problema de maneira elegante.

O que foi que aconteceu aqui? Vamos lá olhar passo a passo da função refatorada.

Primeiro usamos as template strings, o que nos permitiu interpolar as varíaveis de maneira mais elegante.

Segundo, perceba que abrimos a tag de HTML da lista não ordenada e dentro da template string usamos a função map (se você não está familiarizado com a função é só dar uma olhada aqui), mas basicamente o map cria uma nova lista executando alguma operação a cada item. Logo, a cada item da lista projetos eu “tranformo” o nome do projeto em uma string contendo a tag de HTML do item de lista já interpolando com o nome do projeto, dessa forma eu obtenho um novo array com as strings já todas preparadas com a marcação de HTML que preciso.

Porém a função map no final de tudo me dará um novo array com os itens “transformados” no caso do meu exemplo e eu preciso de uma string para montar minha saída de lista não ordenada em marcação de HTML.

Para isso eu uso a função join, que me dará um retorno do array como uma string.

A função join aceita um parâmetro que é o separador, por exemplo, eu poderia pegar um array de lista de frutas e criar um string com cada fruta separada por vírgula se eu passasse uma vírgula como parâmetro.

No entanto eu quero montar uma lista não ordenada de marcação de HTML e quero pegar meu array preparado com os itens da lista (que geramos com a função map) e transformá-lo em uma string sem usar nenhum separador para que não “estrague” minha marcação final de HTML.

Para isso basta passar um espaço vazio como parâmetro de separador para o join.

Dessa maneira eu consegui executar meu loop da lista de projetos manipulando isoladamente cada item, interpolando minha string com a variável que continha o nome do projeto para montar a marcação de HTML desejada.

Agora basta fazer um exercício mental e imaginar as possibilidades dessa abordagem quando você precisar trabalhar por exemplo com marcações de HTML mais complexas como as tabelas. Talvez quem sabe criar seu próprio template engine se você não puder trabalhar com nenhum framework em seu projeto.

Conclusão

Vou ser bastante opinativo aqui, mas eu acho que as template strings são muito úteis e elegantes e tornam seu código mais fácil de ler.

Pode não parecer, mas manipular as strings em javascript pode deixar a leitura do seu código bastante confusa dependendo da complexidade de interpolações que precisam ser feitas, assim sendo as template strings se apresentam como uma solução para trazer um pouquinho de ordem ao caos de aspas e sinais de + que seu código pode se tornar.
